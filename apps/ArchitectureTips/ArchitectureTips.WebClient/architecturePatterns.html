Auto mapping

Role

You need a universal way to adapt one model object to another, for example, you want to create a view model, but do not want to modify the original model class. Similarly you will not break the single responsibility for one or more model classes.

Model-View-View Model

Role

  • The model is the entity that represents the business concept; it can be anything from a simple customer entity to a complex stock trade entity.
  • The View is the graphical control or set of controls responsible for rendering the Model data on screen. A View can be a WPF window, a Silverlight page, or just a XAML data template control.
  • The ViewModel is the magic behind everything. The ViewModel contains the UI logic, the commands, the events, and a reference to the Model.
